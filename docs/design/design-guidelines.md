# Anycast デザイン指示書

## 1. サービス概要

**Anycast** は、AI を活用してポッドキャストを作成・配信できるプラットフォームです。

### ターゲットユーザー

- **リスナー**: AI が作成したポッドキャストを発見・視聴する
- **クリエイター**: AI の力を借りてポッドキャストを制作・配信する

### 参考サービス

- **Spotify**: ダーク UI のトーン、アルバム風カード、ミニプレイヤー、カルーセル表示
- **YouTube**: コンテンツ発見体験、ホームのセクション構成

---

## 2. デザイン方針

| 項目               | 内容                                       |
| ------------------ | ------------------------------------------ |
| **テーマ**         | ダークモード（クールな黒・グレー基調）     |
| **アクセントカラー** | ブルー                                     |
| **雰囲気**         | カジュアル・親しみやすい                   |
| **対応デバイス**   | デスクトップ・モバイル両対応（レスポンシブ） |

## 3. 全体レイアウト構成

```
┌─────────────────────────────────────────────────┐
│  ヘッダー（ロゴ / 検索 / 作成ボタン / アカウント）    │
├────────────┬────────────────────────────────────┤
│            │                                    │
│  サイドバー  │      メインコンテンツエリア          │
│  （常時表示）│                                    │
│            │                                    │
│            │                                    │
├────────────┴────────────────────────────────────┤
│  ミニプレイヤー（固定）                            │
└─────────────────────────────────────────────────┘
```

### デスクトップ

- サイドバー: 左側に常時表示（約 240px 幅）
- ミニプレイヤー: 画面下部に固定

### モバイル

- サイドバー: ハンバーガーメニューで開閉
- ミニプレイヤー: 画面下部に固定（コンパクト表示）

---

## 4. 共通コンポーネント

### ヘッダー

- 左: ロゴ
- 中央: 検索バー
- 右: 「作成」ボタン、アカウントアイコン

### サイドバー

**リスナー向け（メイン）**

- ホーム
- 探索
- ライブラリ（フォロー中、後で聴く、お気に入り、履歴）

**クリエイター向け（Studio）**

- ダッシュボード
- チャンネル
- キャラクター
- BGM
- ボイス

### チャンネルカード

- **形状**: 正方形（アルバムアートワーク風）
- **情報**: サムネイル、チャンネル名、説明（1 行）

### エピソードカード

- サムネイル（正方形 or 横長）
- タイトル
- チャンネル名
- 公開日
- 再生時間

### ミニプレイヤー

- アートワーク（小）
- タイトル / チャンネル名
- 再生コントロール（前へ、再生/一時停止、次へ）
- プログレスバー
- 音量

## 5. 画面別デザイン要件

### 画面 1: ホーム（リスナー向け）

**目的**: ポッドキャストの発見・視聴開始

**レイアウト**:

- 複数セクションを縦に配置
- 各セクションはカルーセル（横スクロール）

**セクション例**:

- 「おすすめチャンネル」- チャンネルカードのカルーセル
- 「最新エピソード」- エピソードカードのカルーセル
- 「人気のポッドキャスト」- チャンネルカードのカルーセル

**インタラクション**:

- カードホバーで再生ボタン表示
- カードクリックで詳細ページへ

---

### 画面 2: Studio チャンネル一覧

**目的**: 自分のチャンネルを管理

**デザインの差別化**:

- メインよりやや落ち着いたトーン（管理画面感）
- 背景色をわずかに明るく、または異なるグレー

**レイアウト**:

- ページタイトル「チャンネル」
- 「新規作成」ボタン（目立つ位置）
- チャンネルカードのグリッド表示

**カード情報**:

- サムネイル
- チャンネル名
- エピソード数
- 最終更新日
- 公開/非公開ステータス

---

### 画面 3: Studio エピソード一覧

**目的**: 特定チャンネルのエピソードを管理

**レイアウト**:

- パンくずリスト（チャンネル > エピソード）
- チャンネル情報サマリー（小さく表示）
- 「新規エピソード作成」ボタン
- エピソードのリスト表示（テーブル or カード）

**リスト項目**:

- サムネイル
- タイトル
- ステータス（下書き / 生成中 / 公開済み）
- 公開日
- 再生数
- アクションメニュー（編集、削除）

---

### 画面 4: Studio チャンネル詳細（台本作成）

**目的**: AI で台本を生成し、エピソードを作成

**レイアウト**:

- ステップインジケーター（進捗表示）
- メインエリア: 台本生成・編集

**台本生成フロー**:

1. **プロンプト入力**: テキストエリアで AI への指示を入力
2. **生成中**: プログレス表示（AI が台本を生成中）
3. **台本表示**: 生成された台本をセグメント単位で表示
   - キャラクター名
   - セリフ内容
   - 編集可能

**UI 要素**:

- 生成ボタン
- 進捗インジケーター（リアルタイム更新）
- 台本プレビュー
- BGM 選択
- 音声合成ボタン
